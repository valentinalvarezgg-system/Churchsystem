# Security Policy

## Supported Versions

The versions currently supported with security updates are:

| Version | Supported          |
| ------- | ------------------ |
| 2.8.x   | :white_check_mark: |
| 2.7.x   | :white_check_mark: |
| 2.6.x   | :x:                |
| < 2.6   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability, please do **not** open a public issue.

Report it privately to:
- `seguridad@churchsystem.com.ar`
- `legal@churchsystem.com.ar` (fallback)

Please include:
- A clear description of the issue
- Steps to reproduce
- Potential impact
- Affected environment/version
- Proof-of-concept (if available)

### Response timeline

- Initial acknowledgment: within **48 hours**
- First triage update: within **5 business days**
- Status updates: at least every **7 business days** until closure

### Disclosure process

- If accepted, we will validate, remediate, and deploy a fix.
- We may request additional details for verification.
- Once fixed, we will publish a security note in the changelog/release notes.
- If declined, we will share a brief rationale.

We appreciate responsible disclosure and collaboration to keep Church System secure.
